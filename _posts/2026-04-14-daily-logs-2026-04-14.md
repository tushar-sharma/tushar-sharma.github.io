## Setup: SUDO_ASKPASS for Non-Interactive sudo

To allow Claude Code to run `sudo` commands without TTY (Touch ID won't work in non-interactive environments), configure an askpass helper:

### 1. Create askpass script

```bash
mkdir -p ~/bin
cat > ~/bin/askpass.sh << 'EOF'
#!/bin/bash
echo "YOUR_PASSWORD_HERE"
EOF
chmod 700 ~/bin/askpass.sh
```

**Security**: Replace `YOUR_PASSWORD_HERE` with your actual password. The file is secured with `chmod 700` (only you can access it).

### 2. Add to `.zshrc`

```bash
export SUDO_ASKPASS="$HOME/bin/askpass.sh"
```

### 3. Add to Claude Code settings

In `~/.claude/settings.json`, add to the `env` section:

```json
"env": {
  "SUDO_ASKPASS": "/Users/YOUR_USERNAME/bin/askpass.sh",
  ...
}
```

And add permissions:

```json
"permissions": {
  "allow": [
    "Bash(sudo -A keytool --import *)",
    "Bash(echo $JAVA_HOME)"
  ]
}
```

**Important**: Use `--import` (double dash), not `-import` (single dash).

## Import Certificate Skill

Add this to `.claude/commands/import-cert.md`:

```markdown
Import a certificate into JVM truststore

Ask me for : 
1. `alias` - a short identifier for this cert (e.g. nexus)
2. `cert_file` - path to the .crt or .pem file to import 

First run `echo $JAVA_HOME` to get the resolved path. Then run:

```
sudo -A  keytool --import -alias <alias> -file <cert_file> -keystore <resolved_path>/lib/security/cacerts -storepass changeit -noprompt
```

Remind me to run `keytool --list -keystore <resolved_path>/lib/security/cacerts | grep <alias>' to confirm it was imported.
```

Restart claude and type `/import-cert`.
