## Promise and Future

I was looking at the code in typescript

```js
async isCustomImage(): Promise<boolean> {
    return ...;
}
```

I wanted to refresh my memory on what a **Promise** was. A Promise is a JavaScript object that represents the eventual result of an asynchronous operation. It may later be fulfilled with a value or rejected with an error.

When people say "future" here, they usually mean the same general idea: a value that is not available yet but will be available later. In JavaScript, the concrete built-in abstraction is `Promise`.


## Jest Testing 

Jest is not only for unit testing. It is commonly used for unit tests, but it can also be used for integration tests, async tests, snapshot tests, and React component tests.

Writing a simple test 

```js

const sum = (a,b) => a + b;

test('add two numbers', () => {
    expect(sum(1,2)).toBe(3);
});
```

Now testing API 

```js
export const fetchUser = async(userId) => {
    const response = await fetch(`https://jsonplaceholder.typicode.com/users/${userId}`); 
    // await pauses this async function until the Promise settles.
    // It does not block the entire JavaScript runtime.

    if (!response.ok) {
        throw new Error('Error');
    }
    return response.json();
}
```

we can write test 


```js
global.fetch = jest.fn(); // mock the fetch function

describe('fetchUser', ()=> {
    it('returns user', async () => {
        const mockUser = { id: 1, name: 'John Doe' };
        global.fetch.mockResolvedValueOnce({
           ok: true,
           json: async () => mockUser,
        });

        const user = await fetchUser(1);
        expect(user).toEqual(mockUser);
        expect(global.fetch).toHaveBeenCalledWith('https://jsonplaceholder.typicode.com/users/1');
    });
});
```

`it` and `test` are aliases in Jest. They do the same thing. People often use `it` because it reads nicely with sentence-style test names, such as `it('returns user')`.

## Mutation Testing

The idea is to intentionally introduce small changes, called mutations, into the code and check whether the test suite catches them. If the tests still pass after a mutation, that usually means the tests are not strong enough in that area.

You can add this to your `Agents.md` file:

`After implementing a feature, run mutation testing when practical to verify that the tests are meaningful, regardless of the language or framework.`
